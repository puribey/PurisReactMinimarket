## Ejercicio Frontend


![alt text](https://github.com/nilusorg/ejercicio-pri/blob/master/interface.png "Cómo se tiene que ver la app")

*Descripción*:

El objetivo de este ejercicio de mostrar una lista de productos e ir cargándolos y eliminándolos de un listado de productos como si fuera un carrito de compras.

Para resolverlo se recomienda usar React y, como no estamos dando archivos maqueados ni un diseño, se pueden usar librerías como Bootstrap u otras para el diseño de la interfaz.

Para que el ejercicio esté completado correctamente se tienen que cumplir todas las siguientes condiciones (sin incluir el bonus):

- [ ] Tiene que mostrar una lista de productos en el `listado de productos` que venga de [este array](https://github.com/nilusorg/ejercicio-pri/blob/master/products.json).

- [ ] Tiene que permitir filtrar la lista de elementos con el `buscador` de la parte superior (tengo que poder escribir `za`, por ejemplo, y en la lista de productos tienen que quedar solo los elementos que en su nombre incluyan `za`).

- [ ] La posibilidad de agregar y eliminar productos en el listado del `pedido` de la derecha. Los elementos se tienen que poder agregar con el botón de `Agregar` en cada producto y eliminar con el botón de `Borrar` en la caja de `Pedido`.


*Bonus*:

- [ ] Mostrar un modal cuando se presiona `completar compra` que le avise al usuario de que la operación terminó exitosamente.


*Qué vamos a observar:*

- Que se resuelva el caso propuesto.

*Qué suma puntos:*

- Que el código sea simple y esté bien comentado en los casos en que sea necesario.
- Que el diseño sea lindo y el maquetado esté bien hecho.

*Qué no es necesario:*

- Usar Redux.
- Que el diseño sea responsive.


### Cómo entregar la solución a este problema:

- Crear un branch en este repo y crear un PR describiendo los cambios que se hicieron.
